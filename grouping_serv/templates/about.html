{% trans "What is the website created for" %}

{% blocktranslate %} I found a problem in real life grouping algorithms. I noticed that most man-made grouping algorithms are not optimized. There are these types of algorithms in common: {% endblocktranslate %}

{% trans "Free grouping:" %}
{% blocktranslate %} In this case, everyone is allowed to choose their groupmates by themselves. Since everyone want to be happy, they choose according to their favorites. However, this caused several groups to be happy while the leftover remainings are unhappy. {% endblocktranslate %}

{% trans "Randomized group" %}
{% blocktranslate %} In this case, the happiness of each group is random, and unable to make the unhappiest group happier. {% endblocktranslate %}

{% trans "Combination Selection Algorithm" %}

{% trans "The website collects data of favored and unfavored groups from each classroom user" %}

{% trans "and form a datastructure: {user: [[favored], [unfavored]]}" %}

{% trans "the program calculates all possibility of grouping of every member in a class" %}

{% trans "Hapiness calculation" %}

{% trans "In each group, if one person is grouped with his/her favored person, overall happiness of the group +1," %}

{% trans "if grouped with unfavored person, overall happiness of the group -1." %}

{% trans "The unhappiest group in each case is measured and stored in a list A," %}

{% trans "the algorithm finally select the case with greatest corresponding value in list A." %}

{% trans "Regression Algorithm" %}

{% trans "The website collects data of favored and unfavored groups from each classroom user" %}

{% trans "and form a datastructure: {user: [[favored], [unfavored]]}" %}

{% trans "The program first create a randomized grouping case, and the program finds the most unhappy group and change the member from the group with other groups randomly. If the score of the most unhappy group exceeds the record, then set it as the current record. If the program cannot beat the record with a fixed attempts, the program will return the record." %}

{% trans "In each group, if one person is grouped with his/her favored person, overall happiness of the group +1," %}

{% trans "if grouped with unfavored person, overall happiness of the group -1." %}

{% trans "Similarities and Differences" %}

{% trans "Similarities" %}

{% trans "1. The parameters of both algorithms are the same" %}

{% trans "Differences" %}

{% trans "1. Since combination selection algorithm would actually enumerate all possibilities of grouping, the grouping process would be significantly slower when class size is about 15. Therefore, the website limits the usage of this algorithm for classes with more than 15 people. However, regression algorithm has fixed complexity, thus it is capable of running in larger classes." %}

{% trans "2. Combination selection algorithm separates the remainders as a new group, but regression algorithm joins the left over members to existing groups. That is because, to improve overall experience while using this website, the computation must be as fast as possible. Since combination selection algorithm has a complexity growing rapidly as class size grows, it puts pressure on the time for computation. If the complexity is further multiplied with the complexity of assigning remaining members to existing groups, it will decrease the user experience and take away the conveniency of the website." %}
